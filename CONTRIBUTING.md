# Contributing to Habit Tracker Prototype

Thank you for your interest in contributing to the Habit Tracker Prototype! We welcome bug reports, feature requests, and pull requests from everyone.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** to your local machine.
3. **Commit your changes**. Please try to make clear, concise commit messages.
4. **Push your changes** to your fork.
5. **Open a Pull Request** against the `main` branch.

## Code Style
- We use Tailwind CSS for styling. Please follow standard utility class usage.
- JavaScript components should use modern ES6 syntax.
- Ensure HTML remains semantically clear and readable.

## Supabase Setup
- Since the project relies on Supabase for Auth and Database, you will need to map your local environment to your own Supabase project url to execute the authentication and database functionality correctly, or utilize the public demo connection string provided if it's open.

Thanks again for helping us make this project better!
